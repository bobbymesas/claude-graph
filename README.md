# Claude Graph

Interactive educational website visualizing Claude Code concepts as a force-directed graph.

## Directory Layout

```
claude-graph/
├── index.html              # Thin HTML shell (body markup + CDN links)
├── styles.css              # All CSS
├── app.js                  # Main logic: physics, rendering, interaction, panel
├── data/
│   ├── graph.js            # Node metadata + edges + CHILDREN + COLORS
│   └── nodes-compiled.js   # AUTO-GENERATED by build.js — do not edit manually
├── content/
│   └── nodes/              # 40 .md files — one per node
│       ├── claude-code.md
│       ├── agent.md
│       └── ...
├── build.js                # Node.js script: content/nodes/*.md → data/nodes-compiled.js
├── vercel.json             # { "buildCommand": "node build.js" }
├── archive/
│   └── claude-concepts-graph-v3.html
└── README.md
```

## Editing Node Content

Each node's description, doc link, and code examples live in `content/nodes/<id>.md`.

Example format (`content/nodes/hook-pretooluse.md`):

```markdown
---
short: Fires before each tool call — can log, validate, or block operations.
doc: https://docs.anthropic.com/en/docs/claude-code/hooks
docLabel: Hooks guide
---

PreToolUse hooks run before Claude executes any tool call...

### EXAMPLE: PreToolUse — validate before Bash calls
### FILE: .claude/settings.json
### LANG: json

\`\`\`json
{ "hooks": { "PreToolUse": [...] } }
\`\`\`
```

After editing, run `node build.js` to recompile.

## Local Development

```bash
node build.js                  # compile content/nodes/*.md → data/nodes-compiled.js
python3 -m http.server 8000    # serve on localhost:8000
```

Open `http://localhost:8000` in your browser.

> **Note:** Must be served over HTTP (not opened as a file) because `app.js` uses ES module imports.

## Deploy

```bash
vercel --prod
```

Vercel runs `node build.js` automatically before deploying (configured in `vercel.json`).
